# Declarative Gameplay Flow Contracts

ThreeNative owns three bounded data-first gameplay contracts for common game
structure:

- `Spawner` is an entity component authored through
  `tn scene set-spawner ... --json`. It owns deterministic prefab spawning,
  wave/interval/once modes, alive/total caps, seeded placement jitter, and
  spawned/depleted observations.
- `GameFlow` is a source document under `content/flow/*.flow.json` authored
  through `tn flow create|add-state|add-transition ... --json`. It owns
  in-scene macro state with closed trigger and action enums.
- `Sequence` is a source document under `content/sequences/*.sequence.json`
  authored through `tn sequence create|add-track|add-key ... --json`. It owns
  fixed-tick camera, transform, event, UI, audio, and timeScale timeline tracks.

These contracts are portable data. Scripts may read state or request playback,
but agents should not hand-roll recurring waves, menu/play/win/fail state, or
cutscene timers before trying these source documents.

## Proof

Current parity evidence is generated by `pnpm verify:conformance`:

- `packages/ir/artifacts/conformance/gameplay-spawner/verification-report.json`
- `packages/ir/artifacts/conformance/game-flow/verification-report.json`
- `packages/ir/artifacts/conformance/sequence/verification-report.json`

Unsupported trigger, action, and track kinds fail closed in IR validation.
Unknown runtime escape hatches remain outside the portable contract.
